# 零信任架构 (Zero Trust)

## 定义

零信任 (Zero Trust) 是现代安全模型，核心原则：**"永不信任，始终验证"** (Never Trust, Always Verify)。

传统安全：防火墙 = 城堡城墙，内外界线分明

零信任：没有内外之分。每次访问都要验证身份、设备和上下文——即使你在"内网"。

## 传统 vs 零信任

| 传统安全 (Castle-and-Moat) | 零信任 (Zero Trust) |
|---------------------------|---------------------|
| 内网 = 安全 | 不信任网络位置 |
| VPN = 进入内网即获所有权限 | 每个资源独立授权 |
| 静态凭据（密码/证书） | 持续验证 + 风险信号 |
| 服务器全可见 | 微隔离 + 最小权限 |

## 核心原则

| 原则 | 含义 |
|------|------|
| **验证一切** | 身份 + 设备健康度 + 上下文 |
| **最小权限** | 只给完成任务所需的最小权限 |
| **假设已被入侵** | 设计上假设攻击者已在内网 |
| **微隔离** | 东西向流量也需身份认证 |
| **持续验证** | 不只是一次登录——每次 API 调用都验证 |

## Google BeyondCorp

Google 是零信任的先驱（2010 年 Aurora 攻击后启动）。

- 员工可在任何网络（咖啡馆/机场）访问内部系统
- 不需要 VPN
- 基于设备证书 + 用户身份 + 设备健康度
- 影响：传统 VPN 在零信任世界中被替代

## 关键技术

| 技术 | 说明 |
|------|------|
| 身份感知代理 (IAP) | 拦截请求，先认证再转发 |
| mTLS (双向 TLS) | 服务间通信互相验证证书 |
| SPIFFE/SPIRE | 云原生身份框架 |
| 设备证明 (Device Attestation) | 验证设备合规性（TPM/安全芯片） |
| 持续访问评估 (CAE) | 实时风险信号触发重新验证 |
| Software-Defined Perimeter | 动态创建 1:1 的连接而非暴露网络 |

## 实施路径

1. **识别数据资产**——哪些要保护
2. **映射数据流**——数据怎么流动
3. **建立身份体系**——SSO + MFA
4. **部署微隔离**——东西向流量控制
5. **启用持续监控**——日志 + SIEM
6. **逐步淘汰 VPN**
