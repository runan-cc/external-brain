# 视频编码基础

## 定义

视频编码是将原始视频数据**压缩为可传输/存储的格式**。视频是数据量最大的媒体类型——没有编码，1 分钟的 4K 视频 ≈ 50GB。编码 = 用算法把 50GB 压到 500MB，人眼看不出区别。

## 原始视频有多大

```
4K (3840×2160) @ 30fps, 8-bit RGB:
3840 × 2160 × 3 bytes × 30 = ~750 MB/s
1 分钟 = 45 GB
1 小时 = 2.7 TB
```

显然不能直接存或传。

## 压缩的核心思想

### 空间冗余（同一帧内）
相邻像素很相似 → 不需要每个像素都存。
- **DCT (离散余弦变换)**：用频率分量表示图像块
- **量化**：丢掉人眼不敏感的高频细节（这里丢的信息人眼看不出来）

### 时间冗余（帧之间）
连续的两帧几乎一样 → 只存差异。
- **I 帧 (Intra)**：完整图像，不依赖其他帧（关键帧）
- **P 帧 (Predictive)**：只存和前一帧的差异
- **B 帧 (Bidirectional)**：参考前后两帧（压缩率最高）

```
GOP (Group of Pictures):
I  B  B  B  P  B  B  B  P  B  B  B  I
↑                    ↑           ↑
关键帧             预测帧      下一个关键帧
```

I 帧 = 最大 (完整图片)
P 帧 = 小 (只存差异)
B 帧 = 最小 (前后参考)

## 编解码器演进

| 编解码器 | 年份 | 压缩率 (vs 原始) | 代表用途 |
|----------|------|-----------------|----------|
| H.261 | 1988 | ~100:1 | ISDN 视频会议 |
| MPEG-2 | 1995 | ~30-60:1 | DVD、数字电视 |
| **H.264 (AVC)** | 2003 | ~100-200:1 | 蓝光、YouTube、监控 |
| **H.265 (HEVC)** | 2013 | ~200-400:1 | 4K UHD 蓝光 |
| VP9 | 2013 | ~200:1 | YouTube (免费版 H.265) |
| **AV1** | 2018 | ~300-500:1 | YouTube/Netflix (免版税) |
| H.266 (VVC) | 2020 | ~500:1 | 8K 广播，刚起步 |

### H.264 vs H.265 vs AV1

| 维度 | H.264 | H.265 | AV1 |
|------|-------|-------|-----|
| 相同质量下的码率 | 基准线 | 少 40-50% | 少 50%+ |
| 编码速度 | 快 | 中 | 慢（改善中） |
| 解码速度 | 最快 | 中 | 中快 |
| 硬件支持 | 所有设备 | 2017+ | 2020+ (较新) |
| 专利 | MPEG LA | 三个专利池 | ✅ 免版税 |
| 浏览器 | ✅ 100% | ~95% (Safari only) | ✅ 95% |

## 关键参数

| 参数 | 含义 | 影响 |
|------|------|------|
| **分辨率** | 像素数 (1080p, 4K) | 越大越清晰 |
| **码率 (Bitrate)** | 每秒比特数 | 决定画质上限 |
| **帧率** | FPS (24/30/60) | 流畅度 |
| **GOP 大小** | 关键帧间隔 | 影响搜索和容错 |
| **Profile/Level** | 编码特性集 | 决定解码器兼容性 |

### 码率参考 (H.264)

| 内容 | 1080p @ 30fps | 4K @ 30fps |
|------|--------------|-----------|
| 静态场景 | 5-15 Mbps | 25-35 Mbps |
| 标准视频 | 25-35 Mbps | 50-60 Mbps |
| 高动作/复杂 | 40-50 Mbps | 100+ Mbps |

## 封装格式 (Container)

| 格式 | 支持编码 | 特点 |
|------|----------|------|
| **MP4** | H.264/H.265/AV1 | 最通用，任何设备都能播 |
| MKV | 所有 | 开源，多音轨/字幕 |
| WebM | VP8/VP9/AV1 | Web 原生 |
| MOV | ProRes/H.264 | Apple 生态 |
| TS | MPEG-2/H.264 | 广播流传用 |

> **MP4 是唯一"到处能放"的格式**——优先使用。

## 实用命令 (FFmpeg)

```bash
# H.264 编码
ffmpeg -i input.mp4 -c:v libx264 -crf 23 -preset medium output.mp4

# H.265 编码（文件减半）
ffmpeg -i input.mp4 -c:v libx265 -crf 28 output.mp4

# 无损裁剪（不重新编码）
ffmpeg -ss 00:01:30 -to 00:02:30 -i input.mp4 -c copy output.mp4
```

CRF 值：0 = 无损, 18-23 = 视觉无损, 28-32 = 有损但可用, 51 = 最差。

## AI 视频生成中的编码

- 几乎所有视频生成模型的训练数据都经过编码（不可能用原始视频）
- H.264 的压缩伪影 = 训练数据的噪声
- **生成的高质量视频也需要编码后才能分发**
- Sora 等模型的输出通常是高质量编码格式
